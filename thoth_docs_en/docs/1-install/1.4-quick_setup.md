# Quick Setup Guide

The **Quick Setup** allows you to get ThothAI running in minutes using the default configuration with the pre-configured `california_schools` database. This is the fastest way to experience ThothAI's capabilities before setting up your own databases and users.

*[Image Placeholder: Quick setup overview diagram]*

!!! tip "Prerequisites"

    Before starting the quick setup, ensure you have:
    - ✅ Completed the [Docker installation](1.2-docker.md) or [local installation](1.3-local.md)
    - ✅ Obtained at least an **OpenRouter API key** with $5-10 credit for testing
    - ✅ Both backend and frontend services running

## Quick Setup Overview

The quick setup uses:
- **Database**: `california_schools.sqlite` (pre-configured education dataset)
- **Default User**: `marco` (requires password setup)
- **AI Provider**: OpenRouter (default configuration)
- **Workspace**: California Schools workspace (pre-configured)

## 2 - Backend Quick Configuration

- **Docker**: http://localhost:8040
- **Local**: http://localhost:8000

Login with the superuser account you created during installation.

### 2.2 - Configure Default User Password

The installation creates a user `marco` without a password. You need to set one:

1. Click the **settings icon** (gear) at the top right to access admin area

*[Image Placeholder: Backend home page with settings icon highlighted]*

2. Navigate to **Authentication and Authorization → Users**

*[Image Placeholder: Django admin interface showing Users section]*

3. Click on user **marco**

*[Image Placeholder: Users list showing marco user]*

4. Set a password (suggestion: `thoth_pwd`) and ensure **Superuser status** is checked

*[Image Placeholder: User edit form with password field and superuser checkbox]*

5. Save the changes

## 3 - Database Preprocessing

Preprocessing prepares the vector database with metadata needed for SQL generation.

### 3.1 - Access Preprocessing Interface

1. Login to backend as user `marco` with the password you just set
2. Navigate to the **Preprocessing** page from the main menu

*[Image Placeholder: Backend home page with Preprocessing menu highlighted]*

### 3.2 - Execute Preprocessing Steps

Execute these three operations in sequence:

*[Image Placeholder: Preprocessing interface showing three buttons]*

#### 3.2.1 - Load Hints
```
Click: "Load Hints"
```
- Loads domain-specific terminology explanations
- Helps AI understand education-specific terms
- Sources data from `california_schools/dev.json`

#### 3.2.2 - Load Questions  
```
Click: "Load Questions"
```
- Loads verified question-SQL pairs
- Provides examples for few-shot learning
- Improves SQL generation accuracy

#### 3.2.3 - Run Preprocessing
```
Click: "Run Preprocessing" 
```
- Processes database schema for vector search
- Creates embeddings for column descriptions
- Enables similarity-based schema linking

*[Image Placeholder: Preprocessing progress indicators]*

!!! info "Column Descriptions"

    You can skip "Update Columns Descriptions" as the default installation already includes AI-generated descriptions for the California Schools database.

## 4 - Generate Database Scope (AI-Assisted)

The database scope helps ThothAI understand if questions are relevant to the database content.

### 4.1 - Access Database Management

## Troubleshooting Quick Setup

### Common Issues

**Frontend won't connect to backend:**
```bash
# Check backend is running
curl http://localhost:8040/api/health/

# Verify DJANGO_API_KEY matches in both _env files
```

**Preprocessing fails:**
```bash
# Check Qdrant is running
curl http://localhost:6333/collections

# Verify API keys are set
# Check container logs for errors
```

**Query execution fails:**
```bash
# Verify database file exists
ls -la thoth_be/data/dev_databases/california_schools/

# Check database permissions
# Review AI provider API key configuration
```
