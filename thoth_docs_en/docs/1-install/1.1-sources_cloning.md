# ThothAI Installation - Getting Sources and Prerequisites

**ThothAI** is a comprehensive AI-powered natural language to SQL conversion system consisting of two main components:

- **Thoth Backend** (Django): Configuration and metadata management
- **ThothSL Frontend** (Streamlit): Natural language to SQL interface

The system can be installed using Docker for production deployment or locally for development. ThothAI provides a unified installation tool that simplifies the entire setup process.

## System Requirements

### Python Requirements
- **Python 3.8 or higher** (Python 3.12+ recommended)
- **pip** package manager
- **Virtual environment** support

## 1 - Getting the Source Code

ThothAI uses a unified repository structure that contains both backend and frontend components along with a sophisticated installation system.

### 1.1 - Clone the Main Repository

Create a directory for your ThothAI installation and clone the main repository:

```bash
mkdir ThothAI
cd ThothAI
git clone https://github.com/mptyl/Thoth.git .
```

*[Image Placeholder: Git clone progress output]*

### 1.2 - Repository Structure Overview

After cloning, you'll have the following structure:

```
ThothAI/
├── install.sh              # Unified installation script
├── installer/              # Installation system
│   ├── install.py         # Main Python installer
│   ├── configs/           # Database and vector DB plugins
│   └── requirements-templates/
├── thoth_be/              # Django Backend
│   ├── install.sh         # Backend-specific installer
│   ├── manage.py          # Django management
│   ├── docker-compose.yml # Backend Docker setup
│   └── installer/         # Backend installation tools
├── thoth_sl/              # Streamlit Frontend  
│   ├── install.sh         # Frontend-specific installer
│   ├── ThothAI.py         # Main Streamlit app
│   ├── docker-compose.yml # Frontend Docker setup
│   └── installer/         # Frontend installation tools
└── thoth_docs/            # Documentation
```

*[Image Placeholder: Directory tree structure]*

!!! note "Why Source-Based Installation?"

    ThothAI uses source-based installation for several reasons:
    
    1. **Customization Flexibility**: Easy modification of components for specific needs
    2. **Environment Configuration**: Templates for environment variables are included with sources
    3. **Docker Integration**: Seamless Docker setup with proper volume bindings
    4. **Open Source License**: MIT license encourages customization and contribution
    5. **Development Support**: Full development environment setup included

## 2 - Installation Methods Overview

ThothAI provides three installation approaches:

### 2.1 - Unified Installation (Recommended)
Use the main `install.sh` script in the project root for complete automated setup:
- Installs both backend and frontend components
- Handles Docker orchestration automatically
- Manages virtual environments
- Configures database and vector database plugins

### 2.2 - Component-Specific Installation
For targeted installations:
- `thoth_be/install.sh` - Backend only
- `thoth_sl/install.sh` - Frontend only

### 2.3 - Manual Installation
For development or custom setups (covered in local installation guide)

### 3.3 - Security Keys Generation

ThothAI requires secure keys for Django and API communication:

#### Django SECRET_KEY
```bash
# Option 1: Online generator
# Visit https://djecrety.ir/

# Option 2: Python command
python3 -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

#### Django API Key
```bash
python3 -c "import secrets; print(secrets.token_urlsafe(32))"
```
